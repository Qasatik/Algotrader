# 🚀 Carry-Bot как SaaS — дорожная карта монетизации

> **Цель:** продавать бота как сервис по подписке (500–2000 ₽/мес) с управлением
> через Telegram, реферальной программой и дополнительными потоками дохода.

---

## ⚠️ Главный принцип: BYOK (Bring Your Own Key)

**Это самое важное решение во всём проекте. От него зависит, законно ли ты работаешь.**

| Модель | Как работает | Законность | Вердикт |
|--------|-------------|-----------|---------|
| **A. BYOK (выбрать эту)** | Пользователь подключает **свой** API-ключ Bybit. Бот торгует на **его** счёте. Ты продаёшь **доступ к софту**. | ✅ Лицензия не нужна (модель software-as-a-service) | **ДЕЛАТЬ ТАК** |
| **B. Пул средств** | Юзеры скидывают тебе деньги, ты торгуешь общим котлом и распределяешь прибыль. | ❌ Незаконно без лицензии управляющего (ДУ). Уголовная ответственность. | **НИКОГДА** |
| **C. Performance fee** | Берёшь % от прибыли юзера. | ⚠️ Серая зона — это уже «доверительное управление». | Избегать |

> **BYOK = ты продаёшь подписку на софт, а не управляешь деньгами.**
> Деньги юзера всегда лежат на **его** бирже, под **его** контролем. Ты не имеешь
> доступа к выводу средств. Это кардинально снижает юридические риски.

Уже реализованный **аудит API-ключа** (P3-13, `core/security.py`) идеально ложится
в эту модель: при подключении бот **принудительно проверяет**, что ключ:
- имеет права только на Spot+Derivatives Trade (без Withdraw/Wallet)
- не read-only
- предупреждает об отсутствии IP-whitelist

## 🏗️ Архитектура: из single-tenant → multi-tenant

Сейчас бот = один процесс, один админ, один аккаунт. Для SaaS нужна
**мультитенантная** архитектура: сотни юзеров, у каждого свой бот.

### Выбор модели изоляции

| Вариант | Изоляция | Сложность | Масштаб | Рекомендация |
|---------|---------|----------|---------|-------------|
| **Процесс на юзера** | Полная | Низкая | ~50 юзеров | Старт (MVP) |
| **Async-задачи в одном процессе** | Средняя | Средняя | ~500 юзеров | Фаза 2 |
| **Docker-контейнер на юзера** | Полная | Высокая | ~1000+ | Фаза 3 (enterprise) |

**Рекомендация для старта:** один Python-процесс с `asyncio`, где каждый юзер —
это независимая корутина `CarryStrategy` со своим конфигом и exchange-клиентом.
База данных (SQLite → PostgreSQL) хранит юзеров, ключи, подписки.

### Схема данных (минимум для MVP)

```
users
├── id (PK)
├── telegram_id (unique)
├── username
├── created_at
├── tier (free / basic / pro)
├── subscription_until
├── referral_code (unique)
├── referred_by (FK users.id)
└── bybit_api_key_encrypted   ← AES-256-GCM, ключ = мастер-секрет из env

subscriptions
├── id (PK)
├── user_id (FK)
├── plan
├── amount
├── paid_until
├── payment_method
└── payment_id

referral_earnings
├── id (PK)
├── referrer_id (FK users.id)
├── referred_id (FK users.id)
├── source (internal / bybit)
├── amount_usdt
└── created_at

bot_configs
├── user_id (FK)
├── symbols / top_n
├── equity_fraction
├── max_notional
├── risk params...
└── enabled (bool)
```

## 🔐 Безопасность (КРИТИЧЕСКИ ВАЖНО)

> API-ключи юзеров = **максимальная ответственность**. Один слив = конец репутации
> и возможные иски.

### Чек-лист безопасности

- [x] **Шифрование ключей at-rest**: AES-256-GCM ✅ (`saas/crypto.py`, commit de8775f).
      Мастер-ключ в env-переменной (никогда в коде/БД). Рассмотреть KMS
      (AWS/GCP/Yandex Lockbox) для прода.
- [x] **Принудительный аудит ключа** при подключении ✅ (`saas/crypto.py` +
      `bot/telegram_saas.py` `/connect`, commit 0036a82): отклоняет ключи с правами
      Withdraw/Wallet/AccountTransfer.
- [ ] **IP-whitelist**: при подключении просить юзера вписать IP сервера в
      настройки ключа на Bybit. Бот проверяет `ips != ["*"]` и предупреждает.
- [ ] **2FA для Telegram** (уже реализовано — TOTP). Привилегированные команды
      (старт/стоп/kill) только после `/auth`.
- [ ] **Rate limiting**: лимит команд на юзера в минуту (anti-spam/abuse).
- [ ] **Логирование действий**: кто, когда, какую команду выполнил (audit log).
- [ ] **Секреты в .env**, `.env` в `.gitignore` (уже есть).
- [ ] **Бэкапы БД** с шифрованием.
- [ ] **Сервер**: только SSH-ключ (без пароля), firewall, fail2ban.

### Что бот НИКОГДА не должен делать
- Запрашивать ключи с правом вывода
- Хранить ключи в открытом виде
- Логировать ключи/секреты
- Иметь общий ключ для всех юзеров

| Шлюз | Комиссия | Плюсы | Минусы |

## 🤑 Потоки дохода (не только подписка)

### 1. Подписка (база) — $3–15/мес × N юзеров
Главный поток. При 100 юзерах (микс Basic/Pro) = ~$400/мес.
При 500 юзерах = ~$2000/мес.

### 2. ⭐ Реферальная программа Bybit (ПАССИВНЫЙ, МАСШТАБНЫЙ)
**Это самый недооценённый источник дохода.**

Bybit платит **до 30% от торговых комиссий** приглашённых пользователей —
**пожизненно**. Carry-бот делает МНОГО сделок (открытие/закрытие/ребалансировка
= тейкер-комиссии). Каждый юзер генерирует комиссии → ты получаешь %.

- Зарегистрируйся в [Bybit Affiliate Program](https://www.bybit.com/affiliate)
- Получи реферальную ссылку
- При онбординге юзера: «Если у вас ещё нет аккаунта Bybit, зарегистрируйтесь
  по нашей ссылке — получите бонусы»
- Доход масштабируется с торговым объёмом юзеров, а не только с подписки

> При 100 активных юзерах, каждый делающих ~$1000 оборота/мес при 0.055% тейкере
> = $55 комиссий/юзер → 30% = $16.5/юзер/мес = **$1650/мес пассивно**,
> поверх подписки.

### 3. Внутренняя рефералка (вирусный рост)
- Каждый юзер получает свой реферальный код
- Приглашённый получает +7 дней триала или -10% на первый месяц
- Пригласивший получает 1 бесплатный месяц за каждого оплатившего реферала
- Это снижает CAC (стоимость привлечения) — юзеры приводят юзеров

### 4. White-label / реселлинг
Продавай платформу другим инфлюенсерам «под их брендом» за % от их подписки
или фиксированную лицензию. Они приводят свою аудиторию, ты даёшь технологию.

### 5. Премиум-стратегии / сигналы
- Базовая carry-стратегия — в подписке
- Дополнительные стратегии (арбитраж, ML-сигналы, grid) — за отдельную плату
- Маркетплейс стратегий в будущем

### 6. Партнёрки (affiliate)
Крипто-аудитория конвертится в:
- VPN-сервисы (для доступа к Bybit из РФ) — % с подписки
- Аппаратные кошельки (Ledger/Trezor) — % с продажи
- Крипто-курсы, сигналы — CPA
- Другие биржи (Binance, OKX affiliate)

### 7. Разовая настройка / onboarding-услуга
«Настроим бота под ключ» — 2000–5000 ₽ разово для тех, кто не хочет разбираться.
Высокий маржинальный доп. доход.

## 📱 Telegram UX — онбординг и управление

### Идеальный flow нового юзера

```
/start
  → Приветствие + описание (1 абзац, что делает бот)
  → /register — регистрация (Telegram ID, язык)
  → Выбор тарифа (inline-кнопки Basic/Pro/VIP + Free Trial)
  → Оплата (или активация триала)
  → /connect — подключение Bybit API:
      1. Инструкция: как создать ключ (только Spot+Derivatives Trade!)
      2. Ввод API key + secret (обрабатывается, шифруется, аудит)
      3. ✅ Ключ проверен: прав достаточно, вывода нет
  → Настройка (equity_fraction, max_notional, top-N) — wizard с дефолтами
  → /start_bot — запуск торгового цикла
  → 🎉 Бот работает, алерты в чат
```

### Команды (расширить текущий telegram_admin.py)

**Бесплатно/все:**
- `/status` — состояние, эквити, открытые позиции
- `/pnl` — P&L за период
- `/stats` — статистика сделок
- `/referral` — мой реферальный код и заработок
- `/help`, `/pricing`

**Подписка:**
- `/connect` — подключить/сменить API-ключ
- `/config` — настройки риска
- `/start_bot`, `/stop_bot` (2FA)
- `/kill` — экстренный флет (2FA)
- `/symbols` — выбор торгуемых пар

### Уведомления (удержание)
- 🟢 Позиция открыта / 🔴 закрыта (уже есть)
- ⏰ Фандинг начислен (раз в 8ч) — показывает заработок
- 📊 Ежедневный отчёт (P&L за день)
- ⚠️ Предупреждения (ликидация, ошибка)
- 🔔 Напоминание об окончании подписки (за 3/1 дня)

> Уведомления = главный фактор удержания. Юзер видит, что бот работает и зарабатывает → не отписывается.

## 📈 Go-to-market: реклама в Telegram

### Подготовка (ДО рекламы — критично!)
1. **Реальный трек-рекорд**: 3–6 месяцев торгуй на своём реальном счёте,
   публикуй честную статистику (P&L, drawdown, win-rate). Без этого реклама
   слита — никто не купит «обещания».
2. **Публичный дашборд**: канал в TG с ежедневными скринами/отчётами бота.
3. **Лендинг**: простая страница (или TG-канал) с тарифами, FAQ, отзывами.
4. **Бесплатный триал**: снижает барьер входа.

### Каналы для рекламы
- **Крипто-каналы РФ** (трейдинг, инвестиции) — закупка рекламы 3000–15000 ₽/пост
- **Каналы про пассивный доход** — твоя ЦА (люди хотят пассивный доход)
- **Чаты Bybit/криптобирж** — нативные упоминания (осторожно с баном за рекламу)
- **YouTube/RuTube обзоры** — договориться с блогерами за % или бартер

### Юнит-экономика (пример)
- CAC (стоимость привлечения): ~$5 (реклама в канале)
- LTV (жизненный цикл): Basic $3 × средние 4 мес = $12 + рефералка
- LTV/CAC = 2.4:1 → выгодно лить рекламу (с учётом апсейла и Bybit affiliate выше)
- **Цель:** конверсия из клика в триал 5–10%, из триала в оплату 15–25%

### Анти-хёрнинг (удержание)
- Триал → быстрая ценность (бот открывает позицию в первый день)
- Ежедневные алерты → юзер помнит о боте
- Реферальная программа → юзер вложился, труднее уйти
- Прозрачность: честная статистика, без обещаний «гарантированных %»

## 🛡️ Юридическая защита и риски

### Обязательные документы
- [ ] **Оферта (Terms of Service)**: чётко — «программное обеспечение,
      не является финансовой консультацией или доверительным управлением».
- [ ] **Раскрытие рисков**: крипта волатильна, возможна потеря средств,
      прошлые результаты не гарантируют будущие.
- [ ] **Политика конфиденциальности** (обработка ПДн — 152-ФЗ РФ / GDPR).
- [ ] **Отказ от ответственности**: юзер торгует на свой риск, на своём счёте.

> В проекте уже есть юридические шаблоны в `qDevelop/legal/` (NDA, DPA,
> Service Agreement, Compliance Hub) — адаптируй их под этот продукт.

### Главные риски
1. **Баг = потеря денег юзера** → даже с офертой, репутационный удар.
   **Митигация:** экстенсивное тестирование, exchange-side SL (уже есть),
   маленькие размеры позиций по умолчанию, kill-switch.
2. **Слив API-ключей** → шифрование, аудит, IP-whitelist.
3. **Bybit блокирует API/меняет правила** → мониторинг, быстрый адаптив.
4. **Регуляторный риск** → BYOK модель минимизирует, но следи за законом
   о крипте в РФ (развивается).

## ✅ Best practices (кратко)

1. **BYOK, не пул средств** — законно и безопасно.
2. **Шифруй API-ключи** как корону — это главная ответственность.
3. **Бесплатный триал** — критичен для конверсии.
4. **Bybit affiliate** — пассивный доход, масштабируется с объёмом.
5. **Реальный трек-рекорд ДО рекламы** — иначе сливаешь бюджет.
6. **Прозрачность** — честная статистика удерживает лучше обещаний.
7. **Уведомления** — юзер должен видеть, что бот работает каждый день.
8. **Самозанятость/ИП** — легализуй приём платежей с первого дня.
9. **Маленькие дефолты** — `max_notional`, `equity_fraction` консервативны,
   чтобы новый юзер не сжёг депозит в первый день (и не обвинил тебя).
10. **Exchange-side SL** — защита работает даже когда бот/сервер упал (уже есть).
